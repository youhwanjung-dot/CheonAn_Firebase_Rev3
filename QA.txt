# TauriApp.tsx 수정 계획

안녕하세요 사장님, QA2.txt 파일 내용을 기반으로 `src/App/TauriApp.tsx` 파일 수정 계획을 수립했습니다. 함께 검토 부탁드립니다.

## 문제점

1.  **잘못된 리소스 경로:** 앱이 초기 `database.json` 파일을 `resources/database.json`에서 찾으려고 하지만, 
실제 파일은 `resources/public/database.json`에 있습니다.
2.  **불분명한 데이터 초기화 과정:** 데이터 파일이 생성될 때, 내용이 비어있는지 혹은 정상적으로 복사되었는지 확인하기 어렵습니다.

## 해결 방안

`src/App/TauriApp.tsx` 파일의 `initializeData` 함수와 `executeReset` 함수를 다음과 같이 수정하여 문제를 해결하고, 데이터 흐름을 명확하게 추적할 수 있도록 하겠습니다.

### 1. 리소스 경로 수정

-   `initializeData` 함수 내의 `path.resolveResource('database.json')` 코드를 `path.resolveResource('public/database.json')`으로 변경합니다.
-   `executeReset` 함수 내의 `path.resolveResource('database.json')` 코드 또한 `path.resolveResource('public/database.json')`으로 함께 변경하여, 데이터 초기화 기능도 정상 동작하도록 수정합니다.

### 2. 단계별 확인 메시지 추가 (`initializeData` 함수)

데이터 파일이 없는 초기 실행 시, 아래의 순서대로 `alert` 메시지를 출력하여 각 단계의 성공 여부와 데이터 상태를 명확히 확인합니다.

-   **1차 확인 (파일 부재 알림):**
    -   `AppData` 폴더에 `database.json` 파일이 없는 것을 확인하는 시점에 알림을 띄웁니다.
    -   `alert('1차 확인: AppData에 파일이 없습니다. 초기 데이터 복사를 시작합니다.');`

-   **2차 확인 (원본 데이터 읽기 검증):**
    -   `public/database.json`에서 원본 데이터를 성공적으로 읽어온 직후, 그 내용을 알림으로 보여줍니다.
    -   `alert(\`2차 확인: 리소스 파일에서 읽어온 원본 데이터:\\n\${resourceFileContent}\`);`

-   **3차 확인 (파일 생성 및 내용 검증):**
    -   읽어온 원본 데이터를 `AppData` 폴더에 `database.json`으로 저장한 직후, 새로 생성된 파일을 즉시 다시 읽어 그 내용을 알림으로 보여줍니다. 이를 통해 파일이 정상적으로 저장되었는지 최종 확인합니다.
    -   `const newlyCreatedContent = await fs.readTextFile(userFilePath);`
    -   `alert(\`3차 확인: 새로 생성된 파일의 내용:\\n\${newlyCreatedContent}\`);`

### 기대 효과

-   `database.json` 파일 경로 오류가 해결되어, 앱 최초 실행 시 정상적인 초기 데이터로 시작할 수 있습니다.
-   단계별 확인 메시지를 통해 데이터 초기화 과정의 투명성을 확보하고, 추후 유사 문제 발생 시 원인 파악을 용이하게 합니다.

위 계획에 따라 수정을 진행하겠습니다.
